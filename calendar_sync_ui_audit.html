Page Layout & Visual Hierarchy

Issue Severity Detail
Flat heading — no visual anchoring High The "Select Calendar Sync method" text is plain fontSize: 24 with no weight, no icon, and no color contrast. It reads like body text rather than a section heading.
No separation between description and tiles Medium The grey description text runs visually into the first tile with only a SizedBox(height: 30). A subtle Divider or a slightly larger gap would create a clear content zone.
Three tiles are visually identical High No icons and no brand colors — Google, Local, and Outlook tiles cannot be distinguished at a glance. Users should recognize each provider instantly.
No scroll protection Medium The body is a plain Column with no SingleChildScrollView. On small screens or with the soft keyboard open, this will overflow and throw a render error.

Calendar Tiles (all three components)

Issue Severity Detail
Raw Border.all() with no color Medium Defaults to black, which clashes in dark themes. Use Border.all(color: Theme.of(context).colorScheme.outline) to respect the active theme.
No provider icon High The biggest visual miss. Each tile needs an icon: the Google "G" logo, Icons.calendar_today for local, and a Microsoft/Outlook icon for Outlook.
Duplicate label text Medium The tile title says "sync with google calendar" and the SwitchListTile inside also says "Sync with google calendars" — the same information is repeated twice inside the same card.
Refresh button inside SwitchListTile title Row Medium Placing an IconButton inside title: is non-standard and visually awkward. Move it to the tile footer as a labelled button (e.g., "Manage →") or as a standalone trailing icon button with a tooltip.
No connection status or account info High After enabling sync, there is no indication of which account is connected. The user cannot tell if the sync is actually working. Show "Connected as user@gmail.com" or a "Not connected" label.
No loading state during async operations High Auth and calendar fetching can take 1–3 seconds. The switch toggles instantly but nothing visible happens. Add a CircularProgressIndicator or disable the switch while the operation is in flight.
Raw exception text shown to user Medium "Error fetching events: $e" dumps a raw Dart exception in a SnackBar. Replace with a friendly message: "Couldn't connect. Check your internet and try again."
Inconsistent tile title casing + typo Low "Sync with local calendar app" vs "sync with google calendar" vs "sync with outtlook calendar" — mixed capitalisation and a double-t typo in "outtlook".

Missing UX Patterns

🔌

Disconnected Empty State

Each tile in its "off" state should show a muted icon and a "Not connected" label so the user immediately understands the current state without interacting.

Impact: High
✅

Connected State Visual

When sync is enabled, show a green badge or colored left-border, plus the account email. Gives the user confidence the feature is active.

Impact: High
🕐

Last Synced Timestamp

"Last synced 5 min ago" builds trust that the sync is functioning correctly, and gives the user a reference point for data freshness.

Impact: Medium
💬

Sync Method Description

A single subtitle per tile — e.g., "Push tasks to Google Calendar as events" — reduces user anxiety about what enabling this actually does.

Impact: Medium
✨

Entry Animations

Tiles appear instantly with no transition. A subtle staggered FadeTransition or SlideTransition on page entry adds polish.

Impact: Low
🎨

Brand Colors per Provider

Use a faint tinted left-border or icon tint per tile (blue for Google, cyan for local, Microsoft blue for Outlook) to create instant visual identity.

Impact: High
