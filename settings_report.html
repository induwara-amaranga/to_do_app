04 — Storage

Persisting Settings via Hive

The app already uses Hive. Add a settings Hive box (it likely already exists in database.dart) and store all settings as key-value pairs. 

// In ToDoDataBase or a dedicated SettingsRepository
class SettingsRepository {
  static final _box = Hive.box('settings');

  // Appearance
  static bool get isDarkMode    => _box.get('darkMode', defaultValue: true);
  static void setDarkMode(bool v) => _box.put('darkMode', v);

  // Notifications
  static bool  get notificationsEnabled => _box.get('notifsEnabled', defaultValue: true);
  static int   get defaultReminderMins  => _box.get('reminderMins',  defaultValue: 30);
  static int   get quietHourStart       => _box.get('quietStart',     defaultValue: 22);
  static int   get quietHourEnd         => _box.get('quietEnd',       defaultValue: 8);

  // Task Defaults
  static String get defaultPriority => _box.get('defaultPriority', defaultValue: 'medium');
  static String get defaultCategory => _box.get('defaultCategory', defaultValue: '');
}
05 — UI Suggestions

Polishing the Settings Experience

Specific, implementable UI improvements for the settings pages and the overall app.

① Grouped tiles with colored icon backgrounds

Each setting tile should have a small rounded square icon with a tinted background (like iOS Settings). You're already doing this in SyncTile — carry the same pattern into settings tiles. Use Container(decoration: BoxDecoration(color: accent.withValues(alpha:0.15), borderRadius: BorderRadius.circular(8))).

② Section headers with subtle top spacing

Add a Padding(padding: EdgeInsets.fromLTRB(16, 20, 16, 6)) section label in ThemeData.colorScheme.primary color with font size 11 and letter spacing 1.5. This breaks long settings pages into scannable groups without a heavy visual divider.

③ Settings hub page — use a Card-per-section layout

On the top-level Settings page, render each category as a Card with a title, subtitle (e.g. "Dark mode · Orange"), and a trailing chevron. This lets users preview their current settings without opening a page.

④ Inline toggles — no separate page for simple booleans

Simple on/off settings (dark mode, vibration, pin starred) should use a Switch directly in the list tile. Only open a sub-page when there are 3+ related options or a complex picker (e.g. quiet hours needs a time range picker).

⑤ Danger Zone with visual separation

Put destructive actions (Clear All Tasks, Reset App Data) in a visually separate section with a red-tinted tile background and a confirmation AlertDialog before executing. Never allow these to run on a single tap.

⑥ Drawer header — show account status

Display the signed-in email below the display name in the drawer header. If not signed in, show "Tap to sign in" as a subtitle. This removes ambiguity about which account is connected.

⑦ Settings search (optional, future)

If the settings grow past 30 items, add a search bar at the top of the hub page. A simple keyword match across setting titles is enough — no fuzzy search needed. Show matched items in a flat list with their section label shown as a subtitle.

⑧ Calendar sync status chips on the hub

On the Calendar & Sync settings tile in the hub, show a small row of colored dots (Google=blue, Outlook=blue, Local=teal) to indicate which sources are connected at a glance, without opening the page.
