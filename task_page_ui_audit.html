Floating Action Buttons

IssueSeverityDetail
Double decoration on FAB Medium Each FAB is wrapped in a Container(color: primary) that also sets a circular BoxDecoration, then the FloatingActionButton itself also sets backgroundColor: primary and a rounded shape. The outer Container is redundant and can cause subtle render clipping.
FABs stacked in a Column High Two FABs are composed inside a Column as the floatingActionButton slot. This is non-standard and loses Material's built-in FAB safe-area, elevation, and positioning behaviour. The standard approach is floatingActionButtonLocation + an ExtendedFloatingActionButton or a SpeedDial package.
Animation placeholder SizedBox(height: 55) High When isDuringAnimation is true the completed-tasks FAB is replaced by an empty SizedBox(height: 55). This leaves a dead gap in the layout and is controlled by a toggled boolean that never resets reliably. Use AnimatedSwitcher or AnimatedOpacity instead.
Warning FAB placed inside Stack not FAB slot Critical The priority warning FloatingActionButton is Positioned(top: 16, right: 16) inside the body Stack. It overlaps the task list content and collides with the real FABs on small screens. It should be shown as a banner, a badge on the main FAB, or a persistent top notification strip.

Sync Progress Indicator

IssueSeverityDetail
Half-width progress bar High The LinearProgressIndicator is constrained to MediaQuery.of(context).size.width * 0.5. On wide screens this looks unfinished; on narrow screens it may misalign. The progress strip should span the full width, like a standard app loading bar at the very top of the screen.
Hardcoded text color Colors.grey[700] Medium The "Syncing X%" label uses a hardcoded grey that is unreadable on dark backgrounds. Use Theme.of(context).colorScheme.onSurface.withValues(alpha: 0.7).
Sync bar overlaps content High The Positioned(top: 0, left: 0) sync bar floats over the task list without any background. Text behind it becomes illegible during a sync. Wrap it in a full-width semi-transparent pill or use a proper LinearProgressIndicator pinned to the SliverAppBar bottom.
No dismiss or cancel affordance Medium During a long sync the user has no way to cancel or dismiss the indicator. At minimum a tap-to-dismiss or a timeout fallback should exist.
SnackBar after sync has no mounted check Critical Line 1303: ScaffoldMessenger.of(context).showSnackBar(...) is called directly after await with no if (mounted) guard. If the user navigated away during the sync, this will throw a BuildContext across async gap error.

Task Group Cards

IssueSeverityDetail
Hardcoded Card(color: Colors.white) Critical Every group card is forced to white regardless of the active theme. This breaks dark mode completely — all cards become blinding white boxes. Replace with Theme.of(context).colorScheme.surface or remove the explicit color and let Material 3 decide.
Raw group key shown as header High groupKey.toUpperCase() displays strings like "2024-11-08" or "2024-11" as group headers. Users see technical date strings instead of human-readable labels like "November 2024" or "This Week". Format these properly with intl DateFormat.
Empty-group "No tasks" has no icon Low A plain centred grey text "No tasks" with no illustration or icon feels sparse. A small icon like Icons.task_alt with a subtle caption improves the empty state.
Redundant "Tasks" sub-label inside each group Low Every expanded group shows a "Tasks" label in the primary colour before the task list. Since the content is obviously tasks, this label adds noise without value. Remove it or replace with a task count badge (e.g., "3 tasks").
Triple repeated if (groupKey == "today") blocks Medium Three consecutive identical if (groupKey == "today" && !showCompletedTasks) blocks map over localCalTasks, googleCalTasks, and outlookCalTasks separately. Merge them into a single loop over all three lists. Also the "Calendar Events" header appears even when all three lists are empty, leaving an orphaned heading.

Warning FAB & Priority Alert System

IssueSeverityDetail
Colors.white used as sentinel value Critical warningColor != Colors.white is the condition for showing the warning FAB. If the app's theme background or card color is white, this comparison breaks, and the warning always shows. Use a nullable Color? warningColor and check warningColor != null.
Priority dialog uses hardcoded light colors High The dialog background is Colors.grey[100], high-priority cards are Colors.red[50], medium cards are Colors.orange[50]. All of these are hardcoded light-mode colors that look wrong on dark themes. Use theme-aware colors or ColorScheme.errorContainer / tertiaryContainer.
Raw date strings in priority dialog subtitle Medium subtitle: Text('Due: ${task[3]} ${task[4]}') shows raw stored strings like "2024-11-08 14:30" rather than a formatted label like "Tomorrow at 2:30 PM". Format with intl before displaying.
Warning FAB has no tooltip Low The warning FAB has no tooltip: parameter. Long-pressing it gives no feedback about what it does. Add tooltip: 'Urgent tasks'.

Missing UX Patterns

📭

Global Empty State

When db.toDoList is empty there is no app-level empty state — the tab bar just shows empty group headers. Show an illustration + "Add your first task" call-to-action when the list is completely empty.

Impact: High
↕️

Pull-to-Refresh for Calendar Sync

Calendar sync only runs at startup via importViewOnly(). There is no way for the user to manually trigger a refresh. Wrap the ListView in a RefreshIndicator to call importViewOnly() on pull.

Impact: Medium
✅

Task Completion Feedback

Completing a task makes it silently disappear from the list. A brief completion animation (slide-out or fade with a check mark) would confirm the action and feel satisfying.

Impact: Medium
🔢

Task Count Badge per Tab

Tabs show only the category name. Adding a badge with the pending task count (e.g., "Work · 4") tells the user what needs attention without switching tabs.

Impact: Medium
🌙

Dark Mode Support

Multiple hardcoded colors (Colors.white, Colors.grey[100], Colors.red[50]) mean the page is effectively light-mode only. A full theme-color audit is needed across cards, dialogs, and indicators.

Impact: Critical
↩️

Undo Task Deletion

Deleting a task is instant and irreversible. A SnackBar with an "Undo" action (holding the deleted task in a local variable for ~5 seconds) is a standard safety net for destructive operations.

Impact: High

Line 688–689 High SortTasksService.sortTasksByMode(db.toDoList, sorting) as List<List<dynamic>>; — the result is cast but never assigned. The sort is silently discarded for the "All" tab, so tasks always appear in insertion order regardless of the selected sort mode.
